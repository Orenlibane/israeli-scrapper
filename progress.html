Remaining Gaps vs PRD
🔴

No user / profile system

The biggest PRD gap. No users table, no search_profiles, no per-user alert delivery. Bot is a broadcast tool. Adding multi-user support requires schema additions (users, search_profiles, alerts) + full onboarding flow.

🔴

Telegram bot commands don't match PRD

PRD specifies /start → onboarding, /myprofiles, /pause, /resume, /delete, /summary, /stop. Built: /search, /top, /stats, /cities, /help. Completely different model — data-query vs profile-management.

🟡

Comparison data quality

Most comparisons fall back to active listing median (not sold transactions) due to sparse nadlan.gov.il data per city+rooms bucket. PRD assumed sold-transaction data would be primary.

🟡

No scoring engine

PRD defines a 0–100 composite score (price/sqm 40%, freshness 20%, price drop 20%, days 10%, feature 10%). Not built — opportunities are sorted purely by pctDiff.

🟡

Missing alert types

"Returned listing" (re-listed after going inactive) and "Long on market" (45+ days) alerts are in the PRD but not implemented. ListingHistory data exists to support both.

⚪

No Grafana / Prometheus

PRD includes observability stack. Currently only Railway logs. Low priority until multi-user scale is reached.

Remaining Gaps
⚠️

Comparison data quality

Comparisons use active listing prices, not sold transactions. The priceNis ≥ ₪50K guard removes most data errors, but extreme outliers can still appear when sparse peer groups skew the median.

⚠️

Neighborhood FK not set by scraper

Scheduled scans don't set neighborhoodId FK on listings (only neighborhoodRaw text). Filter now queries by text match, but FK consistency is missing. Workaround: retroactive backfill job.

⚠️

Single data source

Only Yad2 is scraped. Madlan and other sources would increase coverage and comparison accuracy, especially for sparse city+room combinations.

❓

Hourly digest frequency

Top deals are sent to Telegram every hour. This may be noisy. Consider batching to every 2–4 hours or only sending when new deals appear since last digest.
